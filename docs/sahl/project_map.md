# مشروع جسر المزامنة ونظام طباعة الفواتير
## Sync Bridge & Local Invoice Printing System

**آخر تحديث:** 2026-06-29  
**إصدار المسودة:** v0.1

---

## 1. الرؤية العامة (Vision)

بناء جسر ثنائي الاتجاه بين:
- **برنامج "سهل" المحلي** — قاعدة بيانات MySQL (`sahl2@192.168.0.155:3306`) — نظام ERP ديسكتوب
- **منصة "الأهرام" السحابية** — قاعدة بيانات PostgreSQL (Supabase) — موقع التجارة الإلكتروني و PWA

**الهدف النهائي:** مزامنة الفواتير والعملاء والأصناف تلقائياً، مع توفير واجهة طباعة فواتير محلية تدعم مقاسي A4 و A5، لتكون حلاً انتقالياً ريثما يكتمل التحول السحابي الكامل.

### 2.1 المكونات الرئيسية

| المكوّن | الدور | التقنية |
|---------|-------|---------|
| `sync-bridge` | استخراج البيانات من MySQL وتحويلها وحملها إلى Supabase | Node.js + mysql2 + @supabase/supabase-js |
| `printer-app` | واجهة طباعة فواتير محلية (A4/A5) مع معاينة قبل الطباعة | Electron + React + Tailwind |
| قاعدة البيانات المحلية | MySQL — للسحب فقط (Read-Only Replica) | MySQL 5.7+ (192.168.0.155) |
| قاعدة البيانات السحابية | PostgreSQL — الوجهة النهائية | Supabase (Postgres 15+) |

### 2.2 استراتيجية المزامنة

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   MySQL sahl2 │────▶│ Sync Bridge  │────▶│  Supabase    │
│  (محلي - قراءة │     │ (Node.js)    │     │ (سحابي - كتابة)│
│   فقط)        │     │              │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
                           │
                           ▼
                    ┌──────────────┐
                    │ Printer App  │
                    │ (محلي - طباعة)│
                    └──────────────┘
```

**المراحل:**
1. **Polling:** استعلام دوري (كل 30-60 ثانية) عن الفواتير/العملاء/الأصناف الجديدة أو المعدلة
2. **Transform:** تحويل أنواع البيانات (MySQL types → PostgreSQL types)
3. **Upsert:** إدراج أو تحديث في Supabase باستخدام `ON CONFLICT`
4. **ACK:** تسجيل طابع زمني لآخر مزامنة ناجحة لكل جدول

### 2.3 متطلبات الوصول

| المورد | الحالة | الملاحظة |
|--------|--------|----------|
| MySQL `192.168.0.155:3306` | ❌ غير متاح من JOKER | حساب `root` لا يسمح بالاتصال من هذا المضيف. حلول: (أ) إنشاء مستخدم MySQL مخصص من SERVER-PC، (ب) تشغيل الجسر على SERVER-PC نفسه، (ج) طلب صلاحيات `root@'%'` أو `sync_user@'192.168.0.%'` |
| Supabase (سحابي) | ❌ غير متاح من الشبكة الداخلية | الـ DNS `*.supabase.co` لا يُحل أو متقطع. حل: تشغيل الجسر على خادم خارجي أو استخدام VPN |
| قاعدة MySQL المحلية (`sahl1`) | ✅ متاحة محلياً | لكنها قديمة ولا تحتوي على بيانات `sahl2`. يمكن استخدامها كمرجع مؤقت للهيكل |

## 3. خطة بناء واجهة طباعة الفواتير (A4 & A5)

### 3.1 المتطلبات الوظيفية

| # | المتطلب | الأولوية |
|---|---------|----------|
| 1 | عرض قائمة الفواتير الغير مطبوعة من Supabase | عالية |
| 2 | معاينة الفاتورة (Preview) بحجمها الحقيقي | عالية |
| 3 | طباعة فاتورة فردية على A4 | عالية |
| 4 | طباعة فاتورة فردية على A5 | عالية |
| 5 | طباعة دفعة (Batch) — فواتير متعددة بنقرة واحدة | متوسطة |
| 6 | اختيار الطابعة المحلية (اختيار من قائمة طابعات Windows) | عالية |
| 7 | تخصيص الشعار والهيدر (Company Branding) | متوسطة |
| 8 | معاينة PDF قبل الطباعة | عالية |
| 9 | حفظ الفاتورة كـ PDF للرفع أو الإرسال | متوسطة |
| 10 | وضع عدم الاتصال (Offline) — طباعة من البيانات المخزنة محلياً | متوسطة |

### 3.2 قوالب الطباعة

**A4 (210mm × 297mm):**
- الفاتورة الرسمية الكاملة مع جميع البنود
- خانات: عميل، تاريخ، رقم الفاتورة، البنود (صنف، كمية، سعر، إجمالي)، الضريبة، الخصم، الإجمالي النهائي
- تذييل: شروط الدفع، توقيع المندوب، توقيع العميل

**A5 (148mm × 210mm):**
- فاتورة مبسطة (فاتورة سريعة) للطباعة السريعة أو الإيصال
- نفس البيانات الأساسية بتصميم مضغوط
- مناسبة للطابعات الحرارية الصغيرة (Thermal Receipt)

### 3.3 تقنيات الطباعة المقترحة

| الخيار | الميزة | العيب |
|--------|--------|-------|
| **Electron + printWindow** | تحكم كامل في التخطيط، CSS @page، دعم طابعات Windows | حجم تطبيق كبير نسبياً |
| **React + html2canvas + jsPDF** | خفيف، يمكن تشغيله من المتصفح | دقة طباعة أقل، لا تحكم مباشر بالطابعة |
| **Vite PWA + WebUSB/Bluetooth** | لا يحتاج تثبيت | محدودية دعم الطابعات |

**الاختيار المبدئي:** Electron + printWindow + @react-pdf/renderer لقوالب A4/A5.

### 3.4 تدفق الطباعة

```
فاتورة في Supabase
        │
        ▼
جلب بيانات الفاتورة API
        │
        ▼
عرض في قائمة "فواتير جاهزة للطباعة"
        │
        ▼
اختيار فاتورة → معاينة (Preview)
        │
        ▼
اختيار مقاس (A4 / A5)
        │
        ▼
اختيار طابعة (اختياري)
        │
        ▼
إرسال إلى الطابعة أو حفظ PDF
        │
        ▼
تحديث حالة الفاتورة في Supabase → "تمت الطباعة"
```

## 5. المخاطر والتحديات المعروفة

| المخاطرة | التأثير | خطة التخفيف |
|-----------|---------|-------------|
| ❌ MySQL غير متاح من JOKER | لا يمكن استخراج الهيكل أو المزامنة | طلب إنشاء مستخدم مخصص `sync_user`@`192.168.0.%` |
| ❌ Supabase غير متاح من الشبكة الداخلية | لا يمكن تحميل البيانات | تشغيل الجسر على خادم وسيط (VPS) أو استخدام VPN |
| تغييرات في هيكل MySQL (تحديثات سهل) | كسر الجسر | مراقبة التغييرات مع إنذار عند اختلاف الهيكل |
| حجم بيانات كبير (فواتير قديمة) | بطء المزامنة الأولية | ترحيل تدريجي (Incremental) و batch processing |
| أحرف عربية في MySQL (cp720/charset) | تشويه البيانات | ضبط charset=utf8mb4 في mysql2 |
