**Date:** 2026-12-27 (Updated)
**Status:** Approved vision — guides all future phases

## الرؤية النهائية

نظام الإدارة العليا سيعتمد على **أربع شاشات رسمية فقط** عند اكتمال المشروع:

| # | الشاشة | التركيز | تجيب على السؤال |
|---|--------|---------|-----------------|
| 1 | **النشاط** | مؤشرات النشاط الفعلية (الزيارات، الطلبات، العملاء الجدد، ...) | "ماذا قام الفريق بعمله؟" |
| 2 | **الإنجاز** | النتائج المحققة (المبيعات، نسبة التارجت، التسلسل الهرمي، KPIs) | "ماذا تحقق بالفعل؟" |
| 3 | **الأوزان** | إعدادات فقط — أوزان الـ 6 KPIs وسياسة التقييم | "كيف نحسب الأداء؟" |
| 4 | **التارجت** | إدخال وتعديل الأهداف (شركة ← مديري بيع ← فرق) | "ما المطلوب تحقيقه؟" |

---

## الشاشات الرسمية — التفاصيل

### 1. شاشة النشاط (Activity)

| البند | الوصف |
|-------|-------|
| **المسار** | يُحدد لاحقًا |
| **البيانات** | الزيارات، الطلبات، العملاء الجدد، أي مؤشرات نشاط أخرى |
| **المدى الزمني** | يومي/شهري/مخصص مع فلترة |
| **المستويات** | شركة → مدير → فريق → فرد |
| **المرحلة** | **جديدة — لم تبدأ بعد** |

### 2. شاشة الإنجاز (Achievement)

| البند | الوصف |
|-------|-------|
| **المسار** | يُحدد لاحقًا |
| **البيانات** | المبيعات المحققة، نسبة التارجت، الإنجاز الهرمي، 6 بطاقات KPI |
| **المستويات** | شركة → مدير فريق (مع team_summary) → فرد (مع 6 KPIs) |
| **ملاحظة** | شاشة `HierarchyTargetPage` (`/targets/hierarchy`) هي **النواة الأولية** لهذه الشاشة — سيتم تطويرها لتصبح الشاشة الرسمية |
| **المرحلة** | **النواة موجودة (Phase D) — التطوير الكامل في مرحلة قادمة** |

### 3. شاشة الأوزان (Weights)

| البند | الوصف |
|-------|-------|
| **المسار** | `/dashboard/company-targets` (حاليًا) |
| **البيانات** | أوزان الـ 6 KPIs فقط |
| **ملاحظة** | الشاشة الحالية مناسبة — تحتاج إزالة ملخص الأداء (يخص الإنجاز) |
| **المرحلة** | **موجودة — تحتاج تنظيفًا بسيطًا** |

### 4. شاشة التارجت (Targets)

| البند | الوصف |
|-------|-------|
| **المسار** | يُحدد لاحقًا |
| **البيانات** | هدف الشركة، أهداف مديري البيع، أهداف أفراد الفرق |
| **الميزات** | تدعم التعديل أثناء الشهر حسب قواعد العمل |
| **المرحلة** | **جديدة — لم تبدأ بعد** |

## استراتيجية الاستبدال — Zero Legacy

- لا يوجد مفهوم "Legacy" أو "Deprecated" في النظام النهائي.
- عند اكتمال البديل الجديد، تُحذف الشاشات القديمة **بالكامل** من المشروع.
- لن يُحذف أي شيء إلا إذا كان البديل يؤدي **100%** من وظيفة القديم.

### ملفات الحذف المستهدفة (لكل شاشة)

| الشاشة القديمة | سيحل محلها | الملفات المراد حذفها |
|----------------|:----------:|----------------------|
| Company Targets (حاليًا) | شاشة الأوزان (معدلة) | `CompanyTargetsPage.tsx` |
| Performance Analysis | شاشة الإنجاز | `PerformanceAnalysisPage.tsx` |
| Employee Targets | شاشة التارجت | `EmployeeTargetsPage.tsx` |
| Employee Analysis | شاشة الإنجاز | `EmployeeAnalysisPage.tsx` |
| Team Achievement | شاشة الإنجاز | `TeamAchievement.tsx`, Route `/runtime/achievement` |
| Team Activity | شاشة النشاط | `TeamActivity.tsx`, Route `/runtime/activity` |
| Module Launcher (targets) | يُلغى | القسم الخاص بالأهداف في `ModuleLauncherPage.tsx` |
| HierarchyTargetPage | تندمج في شاشة الإنجاز | `HierarchyTargetPage.tsx` (تستبدل بمكون مدمج) |

### شرط الحذف

لن يُحذف أي ملف حتى يصبح **جميع المستهلكين (importers, routes, navigations)** قد انتقلوا إلى البديل الجديد.

### تقرير الحذف النهائي

بعد اكتمال الاستبدال، يُقدم تقرير يوضح:
- كل شاشة قديمة ومسارها
- كل ملف تم حذفه
- كل Route أُزيل
- ما حل محل كل شاشة قديمة
- تأكيد عدم وجود أي import أو navigate يشير إلى المحذوفات

```
الوضع الحالي ← الوضع النهائي:

  TeamActivity           ─→  شاشة النشاط   (جديدة كليًا)
  TeamAchievement        ─→  شاشة الإنجاز  (تبني على HierarchyTargetPage)
  HierarchyTargetPage    ─→  └─ تندمج في شاشة الإنجاز
  CompanyTargetsPage     ─→  شاشة الأوزان  (تُنظف — تُزال التقارير)
  PerformanceAnalysis    ─→  شاشة الإنجاز  (تندمج)
  EmployeeTargetsPage    ─→  شاشة التارجت  (جديدة كليًا)
  EmployeeAnalysis       ─→  شاشة الإنجاز  (تندمج)
  ModuleLauncher/targets ─→  └─ يُلغى
```

## ملاحظات

3. **الأولوية** — تحدد في بداية كل مرحلة حسب رغبة صاحب المشروع.
