# Upper Management Screens Roadmap (v2)

**Date:** 2026-12-27 (Updated)
**Status:** Approved vision — guides all future phases

---

## الرؤية النهائية

نظام الإدارة العليا سيعتمد على **أربع شاشات رسمية فقط** عند اكتمال المشروع:

| # | الشاشة | التركيز | تجيب على السؤال |
|---|--------|---------|-----------------|
| 1 | **النشاط** | مؤشرات النشاط الفعلية (الزيارات، الطلبات، العملاء الجدد، ...) | "ماذا قام الفريق بعمله؟" |
| 2 | **الإنجاز** | النتائج المحققة (المبيعات، نسبة التارجت، التسلسل الهرمي، KPIs) | "ماذا تحقق بالفعل؟" |
| 3 | **الأوزان** | إعدادات فقط — أوزان الـ 6 KPIs وسياسة التقييم | "كيف نحسب الأداء؟" |
| 4 | **التارجت** | إدخال وتعديل الأهداف (شركة ← مديري بيع ← فرق) | "ما المطلوب تحقيقه؟" |

---

## الشاشات الرسمية — التفاصيل

### 1. شاشة النشاط (Activity)

| البند | الوصف |
|-------|-------|
| **المسار** | يُحدد لاحقًا |
| **البيانات** | الزيارات، الطلبات، العملاء الجدد، أي مؤشرات نشاط أخرى |
| **المدى الزمني** | يومي/شهري/مخصص مع فلترة |
| **المستويات** | شركة → مدير → فريق → فرد |
| **المرحلة** | **جديدة — لم تبدأ بعد** |

### 2. شاشة الإنجاز (Achievement)

| البند | الوصف |
|-------|-------|
| **المسار** | يُحدد لاحقًا |
| **البيانات** | المبيعات المحققة، نسبة التارجت، الإنجاز الهرمي، 6 بطاقات KPI |
| **المستويات** | شركة → مدير فريق (مع team_summary) → فرد (مع 6 KPIs) |
| **ملاحظة** | شاشة `HierarchyTargetPage` (`/targets/hierarchy`) هي **النواة الأولية** لهذه الشاشة — سيتم تطويرها لتصبح الشاشة الرسمية |
| **المرحلة** | **النواة موجودة (Phase D) — التطوير الكامل في مرحلة قادمة** |

### 3. شاشة الأوزان (Weights)

| البند | الوصف |
|-------|-------|
| **المسار** | `/dashboard/company-targets` (حاليًا) |
| **البيانات** | أوزان الـ 6 KPIs فقط |
| **ملاحظة** | الشاشة الحالية مناسبة — تحتاج إزالة ملخص الأداء (يخص الإنجاز) |
| **المرحلة** | **موجودة — تحتاج تنظيفًا بسيطًا** |

### 4. شاشة التارجت (Targets)

| البند | الوصف |
|-------|-------|
| **المسار** | يُحدد لاحقًا |
| **البيانات** | هدف الشركة، أهداف مديري البيع، أهداف أفراد الفرق |
| **الميزات** | تدعم التعديل أثناء الشهر حسب قواعد العمل |
| **المرحلة** | **جديدة — لم تبدأ بعد** |

## الشاشات القديمة — Legacy

تبقى هذه الشاشات موجودة مؤقتًا حتى تكتمل البدائل الجديدة، ثم تُستبدل بالكامل.

**لا يُطوَّر عليها أي ميزات جديدة.**

| الشاشة | المسار | سيحل محلها |
|--------|--------|:----------:|
| Company Targets (النسخة الحالية) | `/dashboard/company-targets` | شاشة الأوزان (معدلة) |
| Performance Analysis | `/dashboard/performance` | شاشة الإنجاز |
| Employee Targets | `/dashboard/employee-targets` | شاشة التارجت |
| Employee Analysis | `/dashboard/employee-analysis` | شاشة الإنجاز |
| Team Achievement | `/runtime/achievement` | شاشة الإنجاز |
| Team Activity | `/runtime/activity` | شاشة النشاط |
| Module Launcher (targets) | `/launcher/targets` | يُلغى |
| HierarchyTargetPage | `/targets/hierarchy` | تندمج في شاشة الإنجاز |

## خريطة الانتقال

```
الوضع الحالي:
  نشاط الشركة (قديم) ─┐
  إنجاز الشركة (قديم) ─┤
  Company Targets     ─┤
  Performance Analysis─┤
  Employee Targets    ─┤
  Employee Analysis   ─┤
  HierarchyTargetPage ─┤ ← نواة أولية
                      └─→ كلها Legacy/انتقالية

الوضع النهائي:
  شاشة النشاط   ← جديدة كليًا
  شاشة الإنجاز  ← تبني على HierarchyTargetPage + توسع
  شاشة الأوزان  ← Company Targets بعد التنظيف
  شاشة التارجت  ← جديدة كليًا
```

## ملاحظات

1. **Collections + Attendance** — قيمها 0/null حاليًا في جميع الشاشات. سيتم تفعيلها في **Phase F**.
2. **لا توجد شاشات أخرى** غير الأربع المذكورة. أي شاشة حالية خارج هذه الأربع هي Legacy.
3. **الأولوية** — تحدد في بداية كل مرحلة حسب رغبة صاحب المشروع.
