# تعديلات النظام — جلسة الهوية والوتساب

## 1. SQL — `20260615_identity_rules_final.sql`

### أ. دور SUPER_ADMIN
- إنشاء دور `SUPER_ADMIN` / `سوبر أدمن` (إن لم يكن موجوداً)
- تعيين أول موظف نشيط كـ SUPER_ADMIN تلقائياً

### ب. `register_customer` — owner_id = super admin
عند تسجيل عميل ذاتياً (`identity_type = 'customer'`)، يكون `owner_id` = أول super admin مباشرة، بدون توزيع عشوائي.

### ج. `get_governed_orders` (plural) — حقول مُثراة
- `customer_maps_url` — رابط خرائط العميل
- `created_by_name`, `created_by_phone`, `created_by_address` — اسم/هاتف/عنوان المرسل
- تحسين `created_by_address` باستخدام `formatted_address` للعملاء

### د. `get_governed_employees` — identity_id
إضافة `identity_id` لكل موظف للسماح بالفلترة والربط مع `orders.created_by`.

### ه. `get_governed_identity` — Helper function جديدة
تحل اسم/هاتف/عنوان لأي `identity_id` سواء موظف أو عميل.

### و. `get_governed_order` (singular) — JSONB مُثرى (جديد)
**المشكلة**: كانت تُرجع جدول `orders` الخام بدون `customer_name`, `owner_name`, `created_by_name` → الوتساب يظهر أقساماً فارغة.

**الحل**: استبدالها بدالة تُرجع JSONB بكل الحقول المثرية:
- `customer_name`, `customer_phone`, `customer_address`, `customer_maps_url`
- `owner_name`, `owner_phone`, `owner_address`
- `created_by_name`, `created_by_phone`, `created_by_address`
- `execution_maps_url` (محسوب من الإحداثيات)
- مع الاحتفاظ بفحص `FORBIDDEN` للموظفين غير المصرح لهم

## 2. Frontend — `src/pages/orders/OrderDetailPage.tsx`

### أ. أولوية العنوان (Fallback Chain)
1. `formatted_address` من `unified_locations`
2. العنوان المدخل يدوياً من `customer_addresses`
3. الاستخراج العكسي عبر `locationService.reverseGeocode(lat, lng)` (Nominatim OSM)
4. الاحداثيات الخام `موقع (lat, lng)` كملاذ أخير

### ب. إعادة هيكلة المتغيرات
- رفع `customerAddress`, `customerMapsUrl`, `customerLocation` إلى نطاق أوسع
- إعادة استخدام بيانات الموقع بين العميل والمرسل (حالة `owner_type = 'customer'`)

### ج. التعامل مع أخطاء RPC
- التحقق من `raw?.error` بعد `get_governed_order` (لأن الدالة الجديدة ترجع JSONB مع مفتاح error بدلاً من رفع استثناء)

## 9. بيانات الطلب الموحدة (Unified Order Display)

### `src/types/order-display.ts`

#### `OrderCustomerData`
```typescript
{
  id: string
  name: string        // customers.company_name
  phone: string       // customer_contacts.phone (primary)
  address: string     // unified_locations.formatted_address → customer_addresses → reverse geocode → site (lat,lng)
  mapsUrl: string     // unified_locations.google_maps_url || https://maps.google.com/?q=lat,lng
}
```

#### `OrderPersonData`
```typescript
{
  id: string
  name: string        // employees.full_name OR customers.company_name
  phone: string       // identities.phone OR customer_contacts.phone
  address: string     // employees.address OR unified_locations.formatted_address
}
```

#### `OrderDisplayData` (المجمّع النهائي)
```typescript
{
  orderId, orderNumber, status, statusLabel, docType, createdAt
  customer: OrderCustomerData       // العميل
  owner: OrderPersonData | null     // المسؤول عن العميل (قد يكون null)
  creator: OrderPersonData          // مرسل الطلب (دائماً موجود)
  creatorType: string               // 'عميل' | 'مندوب مبيعات' | 'موظف'
  items: OrderDisplayItem[]
  itemCount, totalQuantity, totalAmount
  executionLocation: OrderLocationData | null
  notes: string
}
```

#### `buildOrderDisplayData(params)` — المحول الموحد
- يأخذ `order` (من RPC), `items`, `customer`, `owner`, `creator`
- يملأ الحقول مع Fallback: `params.customer?.address || o.customer_address || ''`
- يحسب `creatorType` بناءً على `o.owner_type` و `o.created_by === o.owner_id`
- يحسب `executionLocation` مع `mapsUrl` من الإحداثيات
- يُستخدم في: OrderDetailView, InvoiceView, WhatsApp, PDF

### تدفق البيانات (Data Flow)

```
get_governed_order (JSONB مُثرى)
  ↓
OrderDetailPage.useEffect
  ↓
raw.customer_name / raw.owner_name / raw.created_by_name ← متوفرة مباشرة الآن
  ↓ (مع جلب منفصل للتفاصيل)
customer / owner / creator state objects
  ↓
OrderDetailView props
  ↓
buildFullWhatsAppMessage()  ← تستخدم order.customer_name كـ Fallback
  ↓
WhatsApp message كاملة
```

### مصدر كل حقل (FINAL RULES)

| الحقل | المصدر الوحيد |
|-------|--------------|
| اسم العميل | `customers.company_name` |
| هاتف العميل | `customer_contacts.phone` (primary) |
| عنوان العميل | `unified_locations.formatted_address` → `customer_addresses` → reverse geocode → موقع |
| رابط الخريطة | `unified_locations.google_maps_url` |
| اسم المسؤول | `employees.full_name` (via `customers.owner_id`) |
| هاتف المسؤول | `identities.phone` (via `employees.identity_id`) |
| عنوان المسؤول | `employees.address` |
| اسم مرسل الطلب | `employees.full_name` OR `customers.company_name` |
| هاتف مرسل الطلب | `identities.phone` OR `customer_contacts.phone` |
| عنوان مرسل الطلب | `employees.address` OR `unified_locations.formatted_address` |
| نوع المرسل | `orders.owner_type`: 'customer' أو 'employee' |
